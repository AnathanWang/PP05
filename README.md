# Проектная документация: Информационная система (Авторизация и управление пользователями)

## 1. Описание функционального назначения
Данный модуль является частью информационной системы для предприятия. Он обеспечивает доступ к системе через форму авторизации с проверкой капчи и предоставляет администратору инструменты для управления пользователями.

### Цели и задачи:
- **Аутентификация:** Проверка логина и пароля.
- **Безопасность:** Защита от перебора с помощью пазл-капчи и блокировка при 3 неудачных попытках.
- **Управление пользователями:** Создание, изменение, удаление и разблокировка учетных записей.

## 2. Используемые методы (API)

### Работа с БД (`db.py`)
- `authenticate_user(username, password)`: Аутентификация. Возвращает статус (ok/blocked/reason) и роль.
- `add_user(username, password, role)`: Создание нового пользователя.
- `unlock_user(username)`: Снятие блокировки и сброс счетчика неудачных попыток.
- `update_user(username, password=None, role=None)`: Изменение данных существующего пользователя.

### Интерфейс (`main.py`)
- `LoginWindow`: Основное окно авторизации.
- `UserManager`: Окно администратора для управления пользователями.
- `PuzzleCaptcha`: Виджет интерактивного пазла.

## 3. Отчет о ревью кода (Соответствие ТЗ)

### Соблюдение требований:
- **п. 5.1 - 5.3 (Общие):** БД PostgreSQL, структура таблиц соблюдена.
- **п. 5.4 (Пользователи):** Таблица `users` содержит `login`, `password`, `is_blocked`, `failed_attempts`.
- **п. 5.5 - 5.7 (Авторизация):** Поля обязательны, капча собрана - вход разрешен. Блокировка работает после 3 неудач (пароль или капча).
- **п. 5.8 (Админ):** Реализовано управление пользователями (добавление, редактирование, разблокировка).
- **п. 5.9 (Навигация):** При выходе сессия закрывается, возвращается форма логина.
- **п. 5.10 (UX/Code):** Адаптивный дизайн, русский язык, минимальный размер окон установлен.
